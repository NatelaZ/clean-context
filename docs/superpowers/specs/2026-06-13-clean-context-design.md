# Клинер контекста `/clean-context` — спецификация

**Дата:** 2026-06-13
**Автор:** Натэла + Claude
**Статус:** утверждён дизайн, ждёт вычитки спеки

---

## 1. Проблема

В окружении Claude Code «расход контекста» живёт в двух местах:

1. **Стартовая плата** — фиксируется при запуске сессии: описания скилов, агентов,
   MCP-инструментов, CLAUDE.md, инъекции плагинов. Платится один раз при старте и
   не убирается из живой сессии. Управляется на уровне конфига, *до* запуска.
2. **Растущая плата** — история диалога, копится по ходу. Лечится `/clear`,
   компактом и дисциплиной «одна сессия = одна задача».

Со временем накапливается много включённых, но фактически неиспользуемых скилов,
агентов, MCP-серверов и плагинов. Они раздувают стартовую плату. Нужен инструмент,
который находит лишнее **по факту использования** и обратимо его отключает.

### Что НЕ решаем
- Живой мониторинг расхода контекста в идущей сессии — для этого есть встроенный
  `/context`, повторять хрупко и малополезно.
- Уборку истории диалога — это делает `/clear`/компакт.
- Служебный файл `~/.claude/.last-cleanup` пишет сам Claude Code (уборка истории),
  к нашему клинеру отношения не имеет — не трогаем.

---

## 2. Решение (обзор)

Один скилл-движок `/clean-context`: полный обход конфигурации → оценка стартовой
платы по каждому пункту → сверка с реальной статистикой использования → отчёт →
обратимое применение. Сверху — тонкий планировщик-будильник, дёргающий тот же
движок по расписанию.

**Форма:** скилл-команда как ядро + тонкий планировщик поверх.

**Этапы:**
1. **MVP** — скилл `/clean-context` (компоненты 1–6): ручной аудит и применение.
2. **Поверх** — планировщик-будильник (компонент 7) через существующий нотификатор.

---

## 3. Источники данных (что сканируем)

| Категория | Где лежит активное | Где лежит отключённое | Статистика использования |
|---|---|---|---|
| Скилы | `~/.claude/skills/*/SKILL.md` | `~/.claude/skills-disabled/*` | `~/.claude.json` → `skillUsage` |
| MCP-серверы | `~/.claude.json` → `mcpServers` (глоб.) и `projects.*.mcpServers` | `~/.claude/mcp-disabled/*.json` | — (по факту подключения) |
| Агенты | `~/.claude/agents/*.md` | `~/.claude/agents-disabled/*` (создаём) | косвенно (нет прямой) |
| Плагины | `~/.claude/settings.json` → `enabledPlugins` | там же (`false`) | `~/.claude.json` → `pluginUsage` |
| CLAUDE.md | дерево проектов + `~/.claude/CLAUDE.md` | — | — |
| Hooks | `~/.claude/settings.json` → `hooks` | — | — |

**Формат статистики:** `skillUsage`/`pluginUsage` — объекты вида
`{ "<имя>": { "usageCount": N, "lastUsedAt": <epoch_ms>, ... } }`.

---

## 4. Компоненты

Каждый компонент — отдельный, изолированный, проверяемый блок с понятным входом/выходом.

### 4.1 Сканер инвентаря
**Делает:** собирает полный список всего, что грузится на старте, по всем категориям
из раздела 3.
**Вход:** пути конфигов.
**Выход:** нормализованный список объектов `{ category, name, status (active|disabled),
path, raw }`.

### 4.2 Оценщик стоимости
**Делает:** считает стартовую плату в токенах по каждому пункту.
- Скил/агент: длина поля `description` из frontmatter (именно оно грузится на старте).
- Плагин: сумма стоимостей его скилов.
- MCP: пометка «грузит схемы инструментов» + грубая оценка (в этом окружении MCP
  отложенные/deferred — отмечаем это).
- CLAUDE.md: длина файла целиком (грузится полностью).
**Оценка токенов:** символы → токены с поправкой на язык (≈ /4 для англ., ≈ /2.5 для
кириллицы/смешанного). Метод фиксируем как утилиту, чтобы был один источник правды.
**Выход:** к каждому объекту добавляется `est_tokens`; агрегат «итого стартовая плата»
+ «топ пожирателей».

### 4.3 Сверка с использованием
**Делает:** джойнит инвентарь со `skillUsage`/`pluginUsage`.
**Выход:** к объектам добавляется `usageCount`, `lastUsedAt`, `daysSinceUse`.

### 4.4 Движок рекомендаций
**Делает:** классифицирует каждый пункт.
- **Кандидат на отключение:** `status=active` И (`usageCount=0` ИЛИ `daysSinceUse > 30`).
- **Флаг «не туда»:** `status=disabled`, но `usageCount` высокий (например, `ralph-loop`).
- **Защита:** не предлагать к отключению то, что использовалось в текущей/недавней
  сессии.
**Сортировка:** кандидаты по убыванию `est_tokens` (сначала самые выгодные).
**Выход:** списки `to_disable[]`, `wrong_direction[]`, `keep[]`.

### 4.5 Отчёт
**Делает:** человекочитаемый вывод:
- Общая стартовая плата (токены) и разбивка по категориям.
- Топ пожирателей.
- Рекомендации к отключению с цифрой «вернём ≈ X токенов».
- Предупреждения «не туда».
- Секция «гигиена контекста» (раздел 6).

### 4.6 Применение (безопасно, обратимо)
**Делает:** по явному подтверждению пользователя:
- Скилы/агенты/MCP — **перемещает** в `*-disabled` (никогда не удаляет).
- Плагины — переключает в `settings.json` на `false`.
**Гарантии:**
- Ничего не делает без подтверждения.
- Пишет лог операции (что, откуда, куда, когда) для отката.
- Команда восстановления (обратный перенос / `true`).
**Политика авто-применения:**
- Скилы и MCP — можно применять по подтверждению пачкой (низкий риск, обратимо).
- Агенты и плагины — **только рекомендация**, каждый отключается отдельным явным
  согласием (завязаны на рабочие цепочки: методолог и ко.).

### 4.7 Планировщик-будильник (этап 2)
**Делает:** тонкая обвязка над движком: запускает аудит по расписанию (раз в день
или при старте сессии) и шлёт уведомление в существующий нотификатор
(`~/.claude/hooks/claude-notifier-*`) **только если** `to_disable[]` непуст.
**Не** является отдельным вечным демоном — переиспользует ядро.

## 5. Поток данных

```
Сканер инвентаря
   → Оценщик стоимости (добавляет est_tokens)
      → Сверка с использованием (добавляет usageCount/daysSinceUse)
         → Движок рекомендаций (to_disable / wrong_direction / keep)
            → Отчёт (показ пользователю)
               → [подтверждение] → Применение (перенос в *-disabled, лог)

Планировщик (этап 2): cron/SessionStart → весь конвейер в headless →
   если to_disable непуст → уведомление через нотификатор
```

## 6. Гигиена контекста (стоячие советы в отчёте)

Вторая часть запроса — «что ещё для эффективного расхода окна/лимита». Встраивается
секцией в отчёт:

- **Дисциплина сессий:** одна сессия = одна задача, `/clear` между задачами
  (история — главный пожиратель растущей платы).
- **Тяжёлое чтение — через субагентов:** у них свой контекст, в главное окно
  возвращается только итог.
- **Дробить тяжёлых агентов**, держать описания скилов/агентов короткими — это и есть
  стартовая плата.
- **CLAUDE.md — только необходимое:** грузится целиком и всегда.
- **MCP включать точечно** (у тебя уже аккуратно — активен только `pencil`).

---

## 7. Политика по умолчанию (настраивается)

- «Не используется» = `usageCount: 0` **ИЛИ** `lastUsedAt` старше **30 дней**.
- Авто-применение по подтверждению: скилы, MCP. Агенты, плагины — только рекомендация.
- Не предлагать к отключению использованное в текущей/недавней сессии.
- Все операции обратимы (перенос, не удаление) + лог для отката.

---

## 8. Безопасность и обратимость

- **Никогда не удаляет** — только перемещает в `*-disabled`.
- **Никогда не применяет без подтверждения.**
- **Бэкап конфигов** перед правкой `settings.json` (рядом, как уже делается с
  `claude.json.bak`).
- **Лог операций** для точного отката.
- **Не трогает** служебные файлы Claude Code (`.last-cleanup` и т.п.).
