# Clean Context — клинер контекста для Claude Code

Скилл/плагин, который **аудитит стартовую плату за контекст** Claude Code и
**обратимо** отключает то, что включено, но не используется.

В Claude Code контекст расходуется в двух местах:

1. **Стартовая плата** — фиксируется при запуске сессии: описания скилов, агентов,
   MCP-инструментов, плагинов, содержимое CLAUDE.md, инъекции. Платится один раз и
   управляется на уровне конфигурации, *до* старта.
2. **Растущая плата** — история диалога (лечится `/clear` и компактом).

`clean-context` работает с первой: обходит конфигурацию, оценивает стоимость
каждого пункта в токенах (с опциональной **калибровкой по `/context`** для точности),
**сверяет с реальной статистикой использования** (`skillUsage`/`pluginUsage` из
`~/.claude.json`) и предлагает отключить лишнее.

## Что умеет

- Полный обход: скилы, агенты, MCP-серверы, плагины, CLAUDE.md, hooks.
- Оценка «стартовой платы» в токенах + топ пожирателей и разбивка по категориям.
- **Реальный вес плагинов** — по описаниям их скилов в кеше, а не «ноль».
- **Калибровка оценки по `/context`** — коэффициент реал/оценка по категориям, чтобы
  цифры совпадали с тем, что Claude Code реально показывает (оценка систематически
  занижена; калибровка это убирает). Индикатор калибровки виден в отчёте.
- Рекомендации на основе факта: «включено, но не используется N дней / ни разу».
- Флаги «не туда»: выключено, но активно используется.
- **Разбор CLAUDE.md** по секциям (часто главный пожиратель) + поиск дублей между проектами.
- **Тренды** стартовой платы во времени — снимки копятся сами, видно что выросло/убыло.
- **Обратимое** применение: перенос в `*-disabled` (никогда не удаление), с бэкапами
  и журналом для отката.

## Установка

### Способ A — плагин через маркетплейс (рекомендуется)

В Claude Code:

```
/plugin marketplace add NatelaZ/clean-context
/plugin install clean-context@natela-plugins
```

Затем вызвать: `/clean-context:clean-context` или просто попросить «почисти контекст».

### Способ B — вручную (clone + симлинк)

```bash
git clone https://github.com/NatelaZ/clean-context.git
cd clean-context
bash skills/clean-context/scripts/install.sh
```

Скрипт создаёт симлинк `~/.claude/skills/clean-context` на папку скилла.

## Использование

Попросите Claude «почисти контекст» / «что жрёт контекст», либо вызовите скилл явно.
Он:

1. запустит аудит (только чтение) и покажет отчёт;
2. предложит, что отключить (с пометками `[авто]` / `[спрос]`);
3. по вашему подтверждению отключит выбранное (обратимо);
4. откат — командой restore.

Запустить инструменты напрямую тоже можно:

```bash
node skills/clean-context/bin/audit.js              # отчёт, ничего не меняет
node skills/clean-context/bin/restore.js            # откат последнего применения
node skills/clean-context/bin/claudemd.js           # разбор CLAUDE.md по секциям (только чтение)
node skills/clean-context/bin/trends.js             # динамика стартовой платы по снимкам
```

### Калибровка точности по `/context`

Оценка в токенах — приблизительная и обычно занижена на ~20–25%. Чтобы цифры
совпали с реальностью: откройте `/context`, посмотрите реальные токены категории
(напр. «Custom agents: 2.1k» → `2100`), затем разово выполните:

```bash
node skills/clean-context/bin/calibrate.js agent 2100   # и аналогично: skill <реальные>
```

Коэффициент сохранится в `.state/calibration.json` и применится к будущим аудитам;
в отчёте появится отметка «✓ откалибровано». Повторная калибровка тем же значением
стабильна (идемпотентна) — она всегда считается от сырой оценки.

## Безопасность

- Только перемещение в `*-disabled`, **никогда не удаление**.
- Перед правкой `settings.json`/`.claude.json` создаётся `.bak`.
- Каждая операция журналируется сразу — частичное применение всегда откатывается.
- Служебные файлы Claude Code (`.last-cleanup` и т.п.) не трогаются.
- Изменения вступают в силу с **новой** сессии (стартовая плата фиксируется при старте).

## Разработка

Zero-dependency Node.js (ESM), тесты — встроенный `node --test`:

```bash
cd skills/clean-context
node --test
```

Структура: `lib/` — чистые модули (scan → cost → usage → recommend → report →
apply/restore, плюс `calibration`, `claudemd`, `trends`), `bin/` — тонкие
CLI-обёртки, `test/` — тесты на каждый модуль (100 тестов).
Дизайн и план — в [`docs/superpowers/`](docs/superpowers/).
